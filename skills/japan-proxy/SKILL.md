# 일본 구매대행 스킬 (Japan Proxy Shopping Skill)

이 스킬은 일본 쇼핑몰의 상품 링크를 분석하고, 실시간 환율 및 관세 규정에 맞춰 최종 원화(KRW) 견적을 계산하며, 배송대행지(배대지) 신청서 작성을 돕는 가이드라인을 정의합니다.

---

## 1. 상품 정보 수집 대상 및 규칙 (Target Platforms)

에이전트는 일본 쇼핑몰 URL을 전달받았을 때 다음 정보를 정확히 수집해야 합니다:
- **메르카리 (Mercari)**: 상품명, 가격, 배송비 포함 여부(`送料込み` 여부 확인), 판매자 신용도, 판매 상태(판매 중 / 품절).
- **야후 옥션 (Yahoo Auctions)**: 현재가/즉시구매가, 입찰 수, 남은 시간, 현지 배송비 조건(출하지 및 배송 방법별 요금 확인).
- **라쿠텐 (Rakuten) / 아마존 재팬 (Amazon JP)**: 상품 가격, 일본 내 배송비, 포인트 적립 여부, 재고 상태.

---

## 2. 견적 계산 공식 (Price Calculation)

최종 소비자가 지불할 원화(KRW) 견적은 다음과 같이 계산합니다:

$$\text{최종 견적 (KRW)} = (\text{상품 가격 (JPY)} + \text{일본 현지 배송비 (JPY)} + \text{대행 수수료 (JPY)}) \times \text{적용 환율} + \text{국제 배송비 (KRW)} + \text{관부가세 (KRW)}$$

### 상세 기준:
1. **적용 환율 (Exchange Rate)**: 
   - 실시간 전신환매도율(송금 보낼 때 환율)에 **전산 마진(일반적으로 1% ~ 1.5%)**을 더해 책정합니다.
2. **대행 수수료 (Proxy Fee)**:
   - 기본 수수료: 상품가 5,000엔 미만 시 300엔, 5,000엔 이상 시 면제 (혹은 고정 비율 수수료 적용).
3. **국제 배송비 (International Shipping)**:
   - 무게별 요율표(scripts/calculator.py 또는 resources/shipping_rates.json 참고)를 적용합니다. 해운 특송 또는 항공 특송 기준에 따릅니다.
4. **관부가세 (Customs Duties & VAT)**:
   - **자가사용 목적으로 수입하는 경우, 물품 가격(상품가 + 일본 내 배송비 + 일본 내 세금)이 미화 150달러(USD)를 초과**할 때 과세됩니다. (엔화 기준 고시환율 적용)
   - 관세 = 과세가격 × 관세율 (일반 의류/잡화 약 8%~13%)
   - 부가세 = (과세가격 + 관세) × 10%

---

## 3. 배송대행지(배대지) 신청 양식 (Warehouse Formatting)

구매가 완료되면 에이전트는 사용자가 이용하는 일본 물류창고(배대지) 양식에 맞춰 데이터를 변환해야 합니다.

### 기본 물류창고 정보 (예시):
- **수취인 명**: (배대지 제공 고유 사서함 번호 포함)
- **우편번호**: 540-0001 (오사카 창고 예시)
- **주소**: 大阪府大阪市中央区... [사용자 사서함 번호]
- **전화번호**: 06-xxxx-xxxx

---

## 4. 실행 가이드 (How to Execute)

1. 사용자가 일본 상품 URL을 입력하면, 에이전트는 해당 사이트의 정보를 파악(또는 크롤러 스크립트 실행)합니다.
2. `scripts/calculator.py` 스크립트를 사용하여 환율과 예상 무게를 입력하고 견적을 산출합니다.
3. 산출된 견적서(원화 및 상세 내역)를 사용자에게 마크다운 표로 깔끔하게 정리하여 보고합니다.
4. 사용자가 구매를 확정하면, 현지 주문을 위해 배대지 전송용 JSON 데이터를 생성합니다.

---

## 5. 일본어 번역 및 데이터베이스 저장 (Translation & DB Storage)

일본 쇼핑몰에서 상품 정보를 소싱하여 데이터베이스에 저장할 때, 에이전트는 다음 번역 및 저장 절차를 수행해야 합니다.

### A. 번역 가이드라인
- **상품명 (Title)**: 일본어 고유 명사나 브랜드명은 유지하되, 직관적인 한국어 상품명으로 번역합니다. (예: `【美品】任天堂スイッチ コントローラー` ➔ `[A급/미품] 닌텐도 스위치 컨트롤러`)
- **옵션/색상/사이즈 (Options)**: 바이어나 고객이 쉽게 고를 수 있도록 표준 번역을 적용합니다. (예: `赤` ➔ `레드`, `Mサイズ` ➔ `M 사이즈`)
- **상품 설명 (Description)**: 일본 구매대행 특성상 하자 정보(기스, 작동 여부)가 매우 중요하므로, **작동 여부(동작 확인 완료 여부), 손상(기스, 파손), 구성품 유무**와 같은 핵심 세부사항을 누락 없이 정확한 한국어로 번역하여 정리해야 합니다.

### B. SQLite DB 스키마 구조
에이전트는 `sqlite` MCP 서버를 사용해 다음과 같은 구조의 테이블에 상품을 소싱(저장)합니다.

```sql
CREATE TABLE IF NOT EXISTS sourced_items (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    source_url TEXT UNIQUE,              -- 원본 쇼핑몰 URL
    original_title TEXT,                 -- 일본어 원본 상품명
    translated_title TEXT,               -- 번역된 한국어 상품명
    price_jpy INTEGER,                   -- 원본 엔화 가격
    estimated_krw INTEGER,               -- 환율 및 수수료 계산된 최종 원화 견적
    original_description TEXT,           -- 일본어 원본 설명
    translated_description TEXT,         -- 번역된 한국어 설명
    status TEXT DEFAULT 'Sourced',       -- 상태 (Sourced, Ordered, Shipped 등)
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### C. 에이전트의 역할
1. 소싱 요청을 받으면 `chrome-devtools`를 사용하여 일본 웹페이지 본문을 읽어옵니다.
2. Antigravity의 번역 기능을 활용해 한국어로 매끄럽게 번역합니다.
3. `sqlite` MCP의 `query` 도구를 사용하여 번역된 텍스트와 원본 텍스트를 함께 데이터베이스에 인서트(INSERT)합니다.
